# 24 · M5 抢占节奏 + 到点朝向 + 端到端联调验收 + 排障矩阵

> 目标：把 G1 的「保守串行」升级为「在线抢占」，处理到点朝向，跑通整局，给端到端验收与全局排障表。
> 依赖：[23](23_Nav2耦合_M4_nav2耦合与到达接缝.md) 全部。

## 1. 规划

VLFM 随 `/map` 增长会不断改主意。两种节奏：
- **保守（G1）**：发一个→等 `getResult`→再选。简单，但可能盯着过时 frontier 跑。
- **抢占（G2/G3，推荐）**：~1Hz 重算 `decide_goal`，仅当①检测到目标物体，或②新 best frontier 比当前目标移动超阈值，才 `goToPose` 顶替当前任务。

## 2. 抢占主循环（替换 [23 §3.4] 的内层等待）

```python
PREEMPT_DIST = 0.5      # frontier 移动超过这么多米才换（防抖）
DECIDE_HZ = 1.0
current = None
while rclpy.ok():
    obs = node.build_observation()
    g = policy.decide_goal(obs)
    if g["mode"] == "done":
        if not try_reinspect(): break       # 可选 reinspect 兜底，否则结束
        continue
    if _should_preempt(current, g):
        nav.goToPose(node.to_pose(g)); current = g
        node.publish_goal_marker(g)
    # 内层：边跑边以 DECIDE_HZ 重评估
    t0 = time.time()
    while not nav.isTaskComplete():
        rclpy.spin_once(node, timeout_sec=0.05)
        if time.time() - t0 >= 1.0/DECIDE_HZ:
            t0 = time.time()
            g2 = policy.decide_goal(node.build_observation())
            if _should_preempt(current, g2):
                nav.goToPose(node.to_pose(g2)); current = g2   # Nav2 自动顶替旧目标
                node.publish_goal_marker(g2)
    res = nav.getResult()
    if res == TaskResult.SUCCEEDED:
        v = policy.on_goal_reached(node.build_observation(), current)
        if v["accepted"] and v["next"] == "done":
            node.announce("FOUND & VERIFIED"); break
    elif res in (TaskResult.FAILED, TaskResult.CANCELED):
        policy.on_goal_unreachable(current)
    current = None

def _should_preempt(cur, g):
    if g["mode"] == "done": return False
    if cur is None: return True
    if g["mode"] in ("navigate","navigate-memory") and cur["mode"] == "explore":
        return True                                   # 检测到物体 → 立刻抢占
    if g["mode"] == "explore" and cur["mode"] == "explore":
        return np.linalg.norm(g["xy"] - cur["xy"]) > PREEMPT_DIST   # frontier 漂移够大才换
    return False
```

> 抢占即对 `BasicNavigator` 再次 `goToPose`，Nav2 的 `NavigateToPose` 新目标自动顶替旧目标（无需先 cancel）。`_acyclic_enforcer` 仍在 `decide_goal` 内防横跳，二者叠加更稳。

## 3. 到点朝向（坑：到点对着墙 → 看不到 → 探索卡死）

frontier 目标的意义是「过去看新区域」，到点必须朝**未探索方向**。三选一（按工程量从小到大）：

1. **朝 frontier 方向**（M3 默认 `yaw_hint=_yaw_towards(robot_xy, frontier)`）——简单，多数够用。
2. **朝「frontier→未探索质心」**：在 `ObstacleMap` 加 `frontier_facing_yaw(frontier_xy)`：取 frontier 周围一圈，未探索格（`~explored_area`）的方向均值。更准。
3. **放松 Nav2 yaw 容差**：frontier 目标用大 `yaw_goal_tolerance`（如 π），让 Nav2 不纠结朝向，省事但到点朝向随机。

物体目标（navigate）：`yaw_hint` 朝物体即可，配合较小 `xy_goal_tolerance` 保证看清验证。

**建议**：frontier 用方案 1 + 放松 yaw 容差（方案 3）组合；物体用朝物体 + 小容差。Nav2 参数在 `controller_server` / `goal_checker`：
```yaml
general_goal_checker:
  xy_goal_tolerance: 0.25      # 物体目标
  yaw_goal_tolerance: 3.14     # frontier 目标放松（或为两类目标用不同 goal_checker）
```

## 4. 注意点

- **抢占频率别太高**：`decide_goal` 含 VLM 推理，>2Hz 会拖垮。1Hz 起步，按算力调。
- **建图/感知与发目标解耦**：`build_observation`（含 `/map` 注入）可比 `decide_goal` 更频；但 value/object map 更新跟着 `decide_goal` 走即可。
- **初始 spin**：`nav.spin(spin_dist=2π)` 期间要持续 `spin_once` 泵 RGB-D，让 value map 先铺开，否则第一脚 frontier 质量差。
- **reinspect 兜底**：`mode=="done"` 时可调 `_maybe_reinspect` 的 value 峰值点（[22 §3.4]）再走一程，仍无目标再真正结束。

## 5. 端到端验收（G3：完整一局）

跑一局「找 `<目标物体>`」，全程录 bag + 存 `/vlfm/value_map`、`obstacle_map` 可视化：

| 阶段 | 期望现象 | 判定 |
|---|---|---|
| 启动 | `mission:=false`，VLFM 是唯一发目标者，Nav2 是唯一发 `/cmd_vel` 者 | `ros2 topic info /cmd_vel` 只有 Nav2 发布；`navigate_to_pose` 只有 VLFM 客户端 |
| 初始 | spin 一圈，value map 铺开 | `/vlfm/value_map` 非空 |
| 探索 | 朝 best frontier 走，Nav2 避障，到点朝未探索方向 | 不撞、不对墙卡死、不原地横跳 |
| 发现 | 目标进视野 → mode 切 navigate → 抢占去物体 | 日志 `navigate`，goal marker 跳到物体 |
| 确认 | 到点二次确认（联网 VLM） | 日志 `verify[...] match=...` |
| 拒绝 | 属性不符 → reject 拉黑 → 转探索，不再回该物体 | 日志 `attr mismatch -> reject` |
| 成功 | 属性符合 → 停 + 写记忆 | 日志 `FOUND & VERIFIED`，记忆文件更新 |
| 复用 | 重启同图再跑 → 从记忆 recall | 首个 goal `mode=navigate-memory` |

**硬性指标（无则不算过 G3）**：
- 整局无 `IndexError: edge of map`（越界）崩溃。
- 无「两 frontier 间反复横跳」（acyclic + 抢占阈值生效）。
- 无「到点对墙、value 不增长、探索停滞」。
- `/cmd_vel` 全程仅 Nav2 发布（唯一司机铁律）。
